## 3. Implemented Features

- Email/password register and login with language selection (RU/DE)
- PDF upload: only PDF, max 10 MB, stored in Supabase Storage
- 1 active case per user enforced at DB level (unique partial index) and API level
- Automatic analysis triggered on case page load

- Signed download URLs (1-hour TTL) returned to client
- Final answer screen: displays letter, download buttons for PDF and DOCX
- Full message history stored in `case_messages` (role: ai_question / user_answer)
- Case restores correctly after logout/re-login
- Free/Paid gate: upgrade wall after first completed document
- Upgrade screen: plan selection with simulated payment
- Middleware fix: unauthenticated `/api/*` requests return 401 JSON (not redirect)
- RU and DE UI fully covered with i18n strings

## 5. Free/Paid Implementation Summary

**Flow:**

1. New user: `profile.plan = 'free'`, `free_used = false`
2. First document completes (`status → done`): `free_used` set to `true` via service-role client
3. Next upload attempt: `/api/cases` checks `free_used=true && plan='free'` → returns `402 upgrade_required`
4. Dashboard renders `UpgradeScreen` instead of `UploadForm`
5. User selects plan → `POST /api/upgrade {plan: 'payperuse'|'subscription'}`
6. Profile updated via `createSupabaseAdminClient()` (service role, required due to missing live RLS UPDATE policy)
7. Payment row inserted into `payments` table (simulated)

**Payperuse:** payment inserted per new case (`3.99€`, `type=per_document`, `case_id` set)
**Subscription:** `subscription_active=true`, `subscription_period_start` set, one payment row (`9.99€`, `type=subscription`)

**Note on RLS:** The live Supabase instance is missing the UPDATE policy on `profiles` (migration 001 defines it, but was never applied via `supabase db push`). Workaround: all profile writes use `createSupabaseAdminClient()` (service role key, server-side only).

## 6. AI Workflow Summary

**Analysis phase** (`/api/cases/[id]/analyze`):
1. Download PDF from Storage
2. Extract text with pdf-parse (pdfjs-dist legacy worker)
3. If text < 50 chars → `status=cancelled`, `error_reason=unreadable`
4. GPT-4.1-mini: generate internal analysis summary (Behörde, topic, demands, deadlines, missing info)
5. GPT-4.1-mini (JSON mode via `getNextQaStep`): decide first action
   - `{"action":"question","content":"..."}` → insert `ai_question`, `status=questioning`
   - `{"action":"final","content":"..."}` → generate files, insert result, `status=done`

**Q&A phase** (`/api/cases/[id]/answer`):
1. Insert user answer as `user_answer` message
2. Fetch full message history
3. GPT re-evaluates with analysis summary + full history (JSON mode)
4. Next question → insert `ai_question`, return to client
5. Final answer → upload PDF/DOCX, insert `case_results`, `status=done`, `free_used=true`

**System prompt language:** Questions generated in user's language (RU or DE); final letter always in German (formal, ready to send).

## 7. Storage and Document Generation Summary

**Buckets:**
- `uploads`: `<user_id>/<case_id>/original.pdf` — RLS: owner read/write
- `results`: `<user_id>/<case_id>/result.pdf` + `result.docx` — RLS: owner read/write

**PDF generation** (`generateResultPdf`):
- A4 format (595×842 pt), Helvetica font, 11pt, 16pt line height, 56pt margins
- Word-wrap per line, multi-page support
- Valid `%PDF` header confirmed

**DOCX generation** (`generateResultDocx`):
- One paragraph per line of final text
- Valid PK/zip header, standard OOXML format
- Average size: ~9 KB

**Download:** Signed URLs (1-hour TTL) created server-side via `supabase.storage.createSignedUrl`. Paths stored permanently in `case_results.pdf_path` / `docx_path`; new signed URLs generated on each page load.

## 9. Known Limitations

- **Simulated payments only.** No real payment processor (Stripe, PayPal, etc.). The `payments` table records are inserted without any actual charge. Designed per MVP spec: "Payment is simulated."
- **No subscription renewal logic.** `subscription_period_start` is set on upgrade but no automatic monthly reset or expiry check is implemented.
- **No account management.** No password change, no account deletion, no settings page — per locked MVP rules.
- **PDF font coverage.** Final letter PDF uses Helvetica (Latin-1). Cyrillic characters in final text would not render (final letters are always in German, so this is not an issue in practice).
- **Live RLS UPDATE policy missing on `profiles`.** Migration SQL is correct; policy was not applied via Supabase CLI to the live instance. All profile writes use service-role key as a stable workaround.
- **Signed URL TTL: 1 hour.** If user keeps the result page open longer than 1 hour, download links expire. Re-loading the case page generates new URLs.
